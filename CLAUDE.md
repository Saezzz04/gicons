# gicons

Brand icon monorepo. pnpm workspaces + turbo.

## Commands

- `pnpm install` — install deps
- `pnpm build:svg` — optimize SVGs with SVGO
- `pnpm build:favicons` — fetch real favicons from brand domains (16, 32, 64, 128, 256px)
- `pnpm generate:manifest` — regenerate manifest
- `pnpm generate:react` (vue/svelte/angular/web) — generate components
- `pnpm build` — full build (turbo)
- `pnpm dev` — web dev server (Next.js at apps/web)

## Adding a new brand

When asked to add a brand (e.g., "add Supabase logos"), follow these steps:

### Step 1: Find and download the SVGs
- Search the web for the brand's official logo/press kit (e.g., "Supabase brand assets SVG")
- Common sources: brand's website /press, /brand, /logos page, or GitHub repo
- Download the SVG files (logo, wordmark, mark — in color, white, dark variants)
- If no SVGs are found, ask the user to provide them

### Step 2: Create the brand directory
```
packages/svg/src/brands/{brand-slug}/
```
- Use kebab-case for the directory name (e.g., `supabase`, `visual-studio-code`)
- Place SVG files following naming: `{type}-{variant}.svg`
  - type: `logo`, `wordmark`, `mark`, `icon`
  - variant: `color`, `white`, `dark`, `monochrome`, or brand-specific (e.g., `green`)

### Step 3: Create meta.json
```json
{
  "name": "Brand Name",
  "slug": "brand-slug",
  "category": "devtools",
  "tags": ["relevant", "search", "terms"],
  "website": "https://brand.com",
  "colors": { "primary": "#HEX" },
  "license": "trademark",
  "variants": [
    { "file": "logo-color.svg", "type": "logo", "variant": "color", "background": "light" }
  ]
}
```
Categories: `payments`, `social`, `devtools`, `cloud`, `design`, `communication`, `productivity`, `other`

### Step 4: SVG requirements
- Must have a `viewBox` attribute
- Remove hardcoded `width`/`height` attributes
- No external CSS, fonts, or embedded raster images
- Remove `xmlns` attribute (added during build)

### Step 5: Build
```bash
pnpm build:svg && pnpm build:favicons && pnpm generate:manifest && pnpm generate:react && pnpm generate:vue && pnpm generate:svelte && pnpm generate:web && pnpm generate:angular
```

### Step 6: Verify
- Check that optimized SVGs appear in `packages/svg/dist/brands/{slug}/`
- Check that favicon PNGs appear in `packages/svg/dist/brands/{slug}/favicons/`
- Check that components were generated in `packages/react/src/`
- Run `cd apps/web && npx next build` to verify the website builds

See CONTRIBUTING.md for the complete guide.

## Structure

- `packages/svg/src/brands/` — source SVGs + meta.json per brand
- `packages/svg/dist/brands/` — optimized SVGs (generated by SVGO)
- `packages/svg/dist/brands/{slug}/favicons/` — favicon PNGs (fetched from brand domain)
- `packages/{react,vue,svelte,angular,web}/src/` — framework components (generated)
- `packages/shared/src/` — shared types + manifest (generated)
- `scripts/` — build and generation scripts (TypeScript)
- `scripts/templates/` — Handlebars templates per framework
- `scripts/utils/` — svg-parser.ts, naming.ts
- `apps/web/` — Next.js showcase website

## Conventions

- SVG files: `{type}-{variant}.svg` (type: logo/wordmark/mark/icon)
- Component names: PascalCase `{Brand}{Type}{Variant}` — e.g., `StripeWordmarkBlurple`
- Brand dirs: kebab-case — e.g., `visual-studio-code`
- Always run `pnpm build` after changes to `scripts/` or `scripts/templates/`
- Files in `packages/*/src/` are AUTO-GENERATED — do not edit manually
- Source of truth is `packages/svg/src/brands/` (SVGs + meta.json)
